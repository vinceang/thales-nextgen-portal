Implementation spec

The landing view a passenger sees on connect. Assemble from existing components — no new UI. Everything below renders live to the left; build the React route to match it.

1
App header — NavBar

Props: title="Showcase", logo (airline mark), wifiActive, onMenu, onSearch, onProfile.

Behavior: sticky to top; hamburger opens SideDrawer; search opens the full‑width Search overlay. Height --nav-height (64px).

2
Hero — HeroBanner + CarouselDots

Props: image, kicker="Connect", title (Playfair), ctaLabel="View Plans", onCta.

Behavior: rotating promos; CarouselDots count/active/onSelect drive the slide; CTA routes to Connect/Plans. Auto‑advance ~6s, pause on hover.

3
Category filter — GenrePill row

Horizontally scrollable; one active at a time (active, onClick). The ONE rounded shape in the system — keep it pill.

Data: categories from the catalogue service; selection filters the grids below.

4
Featured rail — TileGrid of ShowcaseTile

TileGrid columns={3} tablet={2} phone={1} gap={16}. Each ShowcaseTile: image, kicker, title, titleSize, onClick.

Loading: render Skeleton variant="block" tiles at the grid's shape while the catalogue loads.

5
Destinations rail — TileGrid (4‑up)

TileGrid columns={4} tablet={2} phone={1} of smaller ShowcaseTiles (titleSize ~20).

Tokens & rules

BG --bg-page · tile titles --font-tile (Noto Serif) · hero headline --font-display (Playfair) · all UI/labels --font-sans. Sharp corners, no shadows, one blue accent. Section gutters --space-lg/--space-xl.

Responsive: TileGrid reflows at 1101 / 561px; type scales fluidly — no manual breakpoints.
