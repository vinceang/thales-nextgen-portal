Connect / Plans

Fidelity Hi‑fi · Desktop 1200px · purchase flow → Modal
Live render — Thales design system

Implementation spec

Plan selection + purchase. The passenger picks a pass, then confirms in a blocking dialog. Use the domain PlanCard as‑is.

1
Header — NavBar + Breadcrumbs

NavBar title="Connect". Below it a Breadcrumbs trail (Showcase › Connect); current page has no href.

2
Intro block

Playfair --font-display headline + muted --font-sans subhead. Plain markup, not a component.

Status: if already connected, show an Alert tone="success" here instead of the headline.

3
Plan grid — PlanCard

Three across (stack on phone). Props per card: name, price, features[], ctaLabel, onSelect. Mark the middle one recommended + badge="Best Value!" (blue top bar, star, filled CTA).

Behavior: onSelect opens the purchase Modal (4) with that plan preloaded. Lay out with TileGrid columns={3} or a flex row of equal flex:1 cards.

4
Purchase confirm — Modal

Modal open title="Confirm Purchase" with footer = secondary Cancel + primary Pay Buttons. Shown here open for reference.

Behavior: Esc / backdrop closes; on success fire a Toast tone="success". Card entry can use the forms controls (see comp 03).

•
Tokens & rules

Recommended accent = --color-highlight-blue; feature checks use the blue check-circle glyph. One hue only — no green "success" card. Sharp corners, no shadows. Equalize card heights with align-items:stretch.
